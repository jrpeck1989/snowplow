In the "SETTINGS" menu, click on "Extensions":

Click "add target":

Choose "HTTP target" from the list of target types to add:

Name the new extension something like "Snowplow Collector - Iglu POST". The "Iglu POST" here represents the fact we will be sending Zendesk events and contexts toÂ Iglu webhook adapterÂ viaÂ POSTÂ request.

In theÂ URLÂ field, enter "https://{{collctor_domain}}/com.snowplowanalytics.iglu/v1?aid=zendesk", replacingÂ {{collctor_domain}}Â with your collector domain.

You can optionally haveÂ ?aid={{my_zendesk_namespace}}Â added to this URL, whereÂ {{my_zendesk_namespace}}Â is a label for the application (here: "zendesk"). This label will be attached to all events fired by the extension, so you can later check where a given event came from (useful if you have more than one Zendesk account).

Set theÂ MethodÂ field to "POST" and theÂ Content typeÂ to "JSON" from the drop-down lists.

Select "Create Target" and click theÂ SubmitÂ button.

We have set up our collector as a Zendesk extension. We can now add a trigger which sendsÂ POSTÂ requests to the collector whenever certain events occur.
